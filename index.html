VUE组件6中通信方式

2019-06-08

vue 组件传值的 6 中方式

总结一下 vue 组件通信的 6 种方式，废话少说,直接开敲

一、props/$emit

​ 1、子组件标签上定义的属性绑定父组件中的数据，然后子组件通过props接收

<!--父组件 -->
<template>
    <div>
        <div @click="change" :style="{'color': color}">parent</div>
        <users :users="users"></users>
    </div>
</template>
export default {
    data () {
        return {
            users: ['join', 'bucky', 'mariny']
        }
    }
}
<!-- users 子组件 -->
<template>
    <div>
        {{users.join(', ')}}
    </div>
</template>
export default{
    props: {
        users: {
            type: Array,
            default: () => []
        }
    }
}

​ 2、子组件通过$emit将事件发送出去，执行父组件中定义的事件

    <!-- 父组件 -->
    <users @test='test' :users="users"></users>
    methods: {
        test (data) {
            console.log('接收到了，数据是:', data)
        }
    }

<!-- users 子组件 -->
<div @click='send'>
    {{users.join(', ')}}
</div>
methods: {
    send () {
        this.$emit('test', 'from users data')
    }
}

二、$emit/$on

​ 俗称公交车bus

​ 1、定义一个js文件，导出一个vue实例 ，子组件通过$emit 发送事件，父组件通过$on 接收

<!--父组件-->
//先导入一个bus文件导出的vue
const Event = () => import('./bus.js')
//在 mounted 里面注册一个监听事件
mounted () {
    Event.$on('send-from', () => {
        this.title = 'from组件修改的title！'
    })
    Event.$on('send-users', () => {
        this.title = 'users组件修改的title!'
    })
}
//在对应的子组件中通过 $emit 发送事件就可以
<!-- from 子组件-->
    //先导入 bus， 通过bus发送
    const Event = () => import('./bus.js')
    methods: {
        test () {
            Event.$em('send-from')
        }
    }
<!-- users 子组件-->
    const Event = () => import('./bus.js')
    methods: {
        test () {
            Event.$em('send-users')
        }
    }

五、provide/inject

​ vue 2.2.0新增的一个api，允许一个祖先组件向其所有子孙后代注入一个依赖，不论组件层次有多深。也就是说，祖先组件通过provide来提供变量，然后子孙组件通过inject来注入变量，主要解决跨级组件的通讯问题。

<!-- parent -->
provide: {
    name: '6月5日到了，祝你生日快乐'
}
<!-- users -->
inject: ['name']

​ 但是这并不是响应式的，要做成响应式的话可以向 provide 提供祖先组件的 this 的，但 this 东西太多，我们采用官方推荐的 Vue.observable 方法

<!-- parent -->
provide () {
    this.theme = Vue.observable({
      color: this.color
    })
    return {
      theme: this.theme
    }
 },
 //修改的时候直接修改this.theme.color就可以了，子孙组建的数据也会变的
<!-- users -->
inject: {
    theme: {
        default: () => ({})
    }
},
//子组件template为functional时推荐用 injections.theme.color

六、$parent/$children 与 ref

​ 1、$parent 取父组件实例， $children 取子组件实例

​ 2、ref如果时标签直接取真实dom节点，如果是组件，返回组建实例

七、总结

​ 父子通信：props 和 $emit、$parent和$children、ref、provide和inject、$attrs和$listeners

​ 兄弟通信：Bus 和 vuex

​ 跨级通信：Bus、vuex、provide和inject、$attrs和$listeners

函数尾调用

2019-05-09

​ 这几天看书的时候遇到一个很有意思的东西，尾递归优化，做笔记来学习一下。

1、什么是尾调用

​ 我们都知道，函数调用时会在内存保留当前的变量、调用位置等信息，我们称为调用帧，当在函数A中调用函数B时，函数A的上层就会形成一个函数B的调用帧，当又在B中调用函数C时，就还有一个C的调用帧，以此类推，一帧一帧的就会形成一个调用栈。

​ 而尾调用的意思顾名思义，那就是 尾部调用，就是说某个函数的尾部去调用另一个函数。

​ 尾调用必须是严格的形式，在函数最后一步执行就可以

2、尾调用的意义

​ 尾调用由于是函数的最后一步操作，所以不需要保存外层函数的调用帧，因为调用位置，内部变量等信息都不会再用到了，直接用内层函数的调用帧取代外层的调用帧就行。

​ 尾调用优化：只保留外层函数的调用帧，如果所有函数都是尾调用，那么完全可以做到每次执行时调用帧只有一项，这将大大节省内存，这就是尾调用的意义

3、尾调用实例

​ 我们都知道，递归是非常消耗内存的，需要保存称成白上千个调用帧，很容易发生栈溢出。我们来看一个递归。

​ 所以，尾调用优化对递归来说意义重大，所以ES6明确规定，所有ECMAScript的实现都必须部署“尾调用优化”。这就是说，再ES6中，只要使用尾递归，就不会发生栈溢出，相对会节省很大内存。

4、对尾调用优化的实现

​ 我学习中遇到一个很有意思的函数，下面就重点详细讲解一下这个函数。

​ 在这个函数中 toc 函数是尾递归优化的实现，变量active在默认情况下是不被激活的，进入尾递归后，这个变量就激活了。然后每一轮sum返回的都是undefined， 所以避免了递归执行，而accumulated数组存放每一轮sum执行的参数，总是有值的，这就保证了accumulator函数内部的while循环总会执行，很巧妙地将“递归”改成了“循环”，而后一轮的参数会取带前一轮的参数，保证了调用栈只有一层。
