base 64

2019-06-23

今天介绍一下base 64，工作中非常常用的一个东西，比如页面开发中像小于10k的图片浪费 http 请求就有点不合适了，不如转 base 64 直接插入页面。一直都停留在下一个插件，自动生成base 64 编码，前几天看 http 权威指南中介绍了一下，写文章总结一下。

这是base 64 编码表

​

下面开始介绍：

​ base 64 编码将一个 8为字节序列拆散成为 6 位的片段，并为每个6位的片段分配一个字符， 这个字符是base 64编码表中的64个字符之一。着64个输出字符都是很常见的，可以安全的放在http首部字段中。base 64 编码用 8 位字符来代表信息中的 6 位，所以base 64 编码字符串大约比原始值扩大了 33% 。

举个例子

一个字符串 ‘Ow!’ 是base 64 编码的，得到的4个字符的base 64 编码之 ‘T3ch’，它是以下面的方式工作的。

  • 字符串 ‘Ow!’ 被拆分成了3个8位的字节 (0x4F, 0x77, 0x21)
  • 这三个字节构成一个 24 位的二进制值010011110111011100100001
  • 这些位被划分为一些 6 位的序列010011、110111、011100、100001
  • 每个 6 位值都表示了从 0~63 之间的一个整数，对应 base 64 字母表中的字符。得到base 64 编码字符串是个4个字符的字符串 ‘T3ch’
8位字符 0 w !
8位值(十六进制) $4F $77 $21
8位值(二进制) 01001111 01110111 00100001
8位值(十进制) 19 55 28 33
base 67 码 T 3 c h

有一些情况下并并不能很好的平均分成6位的块，这时，就在序列的末尾填充零位，使二进制的长度成为24的倍数。这样得到的编码就可能尾部有很多 等号。
